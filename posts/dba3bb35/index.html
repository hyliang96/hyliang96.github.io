在服务器和本机上通用的Vim配置

先安装依赖配置，以下依赖文件可直接复制到~/.vimrc文件（若无该文件，则vim ~/.vimrc创建该文件）

不用激活该文件，直接打开vi或vim，设置即可生效

" :Qargs 将quickfix 列表中的文件加入 args 列表中
command! -nargs=0 -bar Qargs execute 'args' QuickfixFilenames()
function! QuickfixFilenames()
  " Building a hash ensures we get each buffer only once
  let buffer_numbers = {}
  for quickfix_item in getqflist()
    let bufnr = quickfix_item['bufnr']
    " Lines without files will appear as bufnr=0
    if bufnr > 0
      let buffer_numbers[bufnr] = bufname(bufnr)
    endif
  endfor
  return join(map(values(buffer_numbers), 'fnameescape(v:val)'))
endfunction

  • tmux与vim配色冲突：在tmux里进入vim，文字自带不透明纯黑底色，为无文字处的底色是dark色。为了使得文字也变成透明底色，需设置vim ~/.vim/colors/molokai.vim ，将全文中，所有hi Normal ... ctermbg=xxx改为hi Normal ... ctermbg=NONE 。这样修改，对macvim没有干扰。

从服务器VIM 复制/剪切到本机系统剪贴板

参考

安装clipper

  • 在个人电脑 安装clipper，MacOS使用 brew install clipper 安装

  • 用brew将clipper为开机自启 brew services start clipper

  • 检查clipper否正常启动

ps -ef | grep clipper
# 返回 502   463     1   0  3:07PM ??         0:00.09 /usr/local/opt/clipper/bin/clipper

简单演示

  • clipper启动后，通过下面的命令连接到远程机器：
ssh 用户名@网址或ip -R 8377:localhost:8377

-R 服务器端口:localhost:本机端口其含义是在ssh时，将此服务器端口将转发到此本地端口。

clipper默认从8377读取远程传来的文本，将其写入到本机系统剪切板。

  • 确认远程机器上安装有 nc
nc
# 返回
# usage: nc [-46AacCDdEFhklMnOortUuvz] [-K tc] [-b boundif] [-i interval] [-p source_port] # [--apple-delegate-pid pid] [--apple-delegate-uuid uuid]
#     [-s source_ip_address] [-w timeout] [-X proxy_version]
#     [-x proxy_address[:port]] [hostname] [port[s]]
  • 简单演示
echo hello,world | timeout 0.1 nc localhost 8377 # 服务器端口

则 “hello,world” 字符转已经成功复制到本机的剪贴板了，可以command+v贴出。

  • timeout 0.1表示0.1秒内发出去，超时就结束连接，不管对面收没收到

若不加此参数，在我用的nc版本

OpenBSD netcat (Debian patchlevel 1.187-1ubuntu0.1)

命令行下敲完echo hello,world | nc localhost 8377 ，会一致等待，而不允许输入下一行命令，除非ctrl+C结束。

  • 测试：在本地开此监听程序
nc -lk 8377

-k: 持续监听。若不带此参数，听到一条消息就立即退出。若带此参数，不论听到多少条消息都不退出，需要ctrl+c退出。

-l: 监听传入消息。若不带此参数，就表示传出消息。

若在服务器上执行echo hello,world | timeout 0.1 nc localhost 8377时，本地的监听程序返回“hello world”，则说明 【服务器8377端口→ssh -R→本地8377端口 】这种转发信道可用。

  • 配置ssh设置

为免去每次登录加 -R 8377:localhost:8377 参数，可修改 ~/.ssh/config 文件，其中每个Host配置下加一句RemoteForward 8377 localhost:8377（RemoteForward 服务器端口 localhost:本机端口）。完整的一个Host配置例如：

Host 别名
    HostName 网址或ip
    Port 端口号
    User 用户名
    IdentityFile ~/.ssh/id_rsa
    RemoteForward 8377 localhost:8377

在远程主机的VIM里复制内容

  • 我们可以利用 :[range]w[rite] [++opt] !{cmd} 来实现文件内容的复制：
    • 复制当前行：在 VIM 中执行 :.w !nc localhost 8377
    • 复制选中的行：选中要复制的行后，执行 :'<,'>w !nc localhost 8377 ( vy选中后，进入命令行模式后 VIM 会自动帮忙填充 '<,'>)
    • 复制整个文件：在 VIM 中执行 :%w !nc localhost 8377
    • 复制 VIM 默认 register 的内容：在 VIM 中执行 :call system('nc localhost 8377', @")
  • 为方便起见，在~/.vimrc加入
" 从服务器的vim里复制/剪切文字到本地电脑的剪切板
" v,y复制/v,d剪切/(数字)dd 整行剪切，发送到笔记本的系统剪切板，但不清空vim的寄存器
vmap y y:call system('timeout 0.1 nc localhost 8377', @")<CR>
vmap d d:call system('timeout 0.1 nc localhost 8377', @")<CR>
nmap dd dd:call system('timeout 0.1 nc localhost 8377', @")<CR>
" ctrl+a 整个文件发送到笔记本的系统剪切板
map <C-a> :%w !timeout 0.1 nc localhost 8377<CR><CR>

其中@"表示vim的"剪切板中的内容，在原生的vim中，每次按y yy d dd c cc操作，都会写到这个剪切板
