自定制终端登录界面

用sudo创建可执行文件/etc/update-motd.d/<执行序号>-文件名（bash脚本），则此脚本会在ssh登录时执行，脚本输出会显示在ssh登录界面上。此法可用户定制酷炫的ssh登录界面，例如

安装方法

安装依赖

  • lolcat 让终端输出彩虹色

  • figlet 让终端创建字符图案

sudo apt install lolcat figlet

配置开机输出-hostname

预期效果

tmp='/etc/update-motd.d/10-hostname-color'
sudo touch $tmp
sudo chmod a+x $tmp
sudo vim $tmp

vim中写入

#!/bin/bash

/usr/bin/figlet "$(hostname)" | /usr/games/lolcat -f

配置开机输出-磁盘使用

预期效果

tmp='/etc/update-motd.d/35-diskspace'
sudo touch $tmp
sudo chmod a+x $tmp
sudo vim $tmp

vim中写入

#!/bin/bash

# config
max_usage=90
bar_width=50
# colors
white="\e[39m"
green="\e[1;32m"
red="\e[1;31m"
dim="\e[2m"
undim="\e[0m"

# disk usage: ignore zfs, squashfs & tmpfs
mapfile -t dfs < <(df -H -x zfs -x squashfs -x tmpfs -x devtmpfs --output=target,pcent,size | tail -n+2)
printf "\ndisk usage:\n"

for line in "${dfs[@]}"; do
    # get disk usage
    usage=$(echo "$line" | awk '{print $2}' | sed 's/%//')
    used_width=$((($usage*$bar_width)/100))
    # color is green if usage < max_usage, else red
    if [ "${usage}" -ge "${max_usage}" ]; then
        color=$red
    else
        color=$green
    fi
    # print green/red bar until used_width
    bar="[${color}"
    for ((i=0; i<$used_width; i++)); do
        bar+="="
    done
    # print dimmmed bar until end
    bar+="${white}${dim}"
    for ((i=$used_width; i<$bar_width; i++)); do
        bar+="="
    done
    bar+="${undim}]"
    # print usage line & bar
    echo "${line}" | awk '{ printf("%-31s%+3s used out of %+4s\n", $1, $2, $3); }' | sed -e 's/^/  /'
    echo -e "${bar}" | sed -e 's/^/  /'
done

试运行-最终效果

运行登录输出的出现

sudo run-parts /etc/update-motd.d

如得到上述输出，则配置成功。之后每次ssh <服务器>的截面均是上图输出。注：非ssh登录，则不会有此输出，例如当ssh登录后运行bash，则无此输出。

开机过程中

alt+ctrl+del：在系统启动完成前，按此可重启系统，而无需去按电源键 参考

BIOS 界面

esc: 一开机/重启, 就持反复之, 直到首次出现供应商的页面(如超威/斯腾合力), 而后会进入 BIOS 界面:

.

在 BIOS 界面中的操作:

  • 可以配置 IPMI 等内容
  • 配置完成, 按 esc 选 yes 退出 BIOS 界面

grub 界面

从 BIOS 界面退出会进入 Grub 界面:

  • 若选 Ubuntu 则正常开机

  • 若选 Advanced options for Ubuntu, 回车, 则进入如下界面

此界面中若选 recovery mode, 回车, 则进入恢复模式界面

恢复模式

恢复模式界面如图:

若选 root, 回车, 则其下方出现 root 终端, 可再此输入命令, 如修改用户密码/root 密码/修改网络配置/修改防火墙配置 等等

完成后, 在下方终端 exit, 而后弹出对话框中选 yes (or ok), 而后会根据修改后的配置, 继续正常开机

ISO介绍

xxxx.iso文件，称镜像（mirror）文件，相当于光盘，内存有文件。

安装ISO中的文件

挂载镜像

sudo mkdir /mnt/<挂载文件夹名>
sudo mount -o loop <xxx>.iso /mnt/<挂载文件夹名>

注：只有root用户才能执行mount命令

参数

  • -t iso9660 -t（文件系统类型）= iso9660（CD-ROM光盘标准文件系统），此参数可选，非必需
  • -o loop -o （选项） loop （用来把一个文件或着镜像当成分区挂上系统）

安装

cd /mnt/<挂载文件夹名>

可以看见其下的文件，而后按照当中的文件即可，如./<可执行安装文件> ，或 自己 make 安装，等等

解挂载

sudo umount <挂载路径>
sudo rm -rf <挂载路径>

从宇宙繁衍到科研的意义

发表于 2021-08-02 01:24:20 更新于 2021-08-02 01:27:31 分类于 道 Waline：
本文字数： 1.3k 阅读时长 ≈ 4 分钟

背景

最近一年感到压力很大啊，又看到身边的人同样各有各的苦。众生皆苦呀，说的没错。顺势看了些佛教的东西，可是我还未能接受轮回的观点，但却自以为有了出离心，觉得 “生是一切苦的起源” “一切诸行无常，一切法无我，涅槃寂灭”好有道理诶。我意识到，要是没有轮回这个设定，那谁死都是涅槃了咯，那我不用活下去了。于是赶紧找找存在的意义让自己坚持活下去。

宇宙的历程

一次偶然，看了一个讲宇宙生命历程的视频，让我闪现了一种答案来回答存在的意义。这个视频是目前天体物理学对宇宙演化的一种预测，好几个画面让我感到十分恐惧：

  • 宇宙中所有恒星熄灭后，幸存的只能在散发余热的白矮星旁苟活。他们绕着白矮星修建了戴森环，以收集微弱的能量，直到白矮星最后一丝余热散近。

  • 只剩下黑洞的僵尸宇宙中，黑洞构成星系，它们相互合并释放巨大的能量，最终黑洞也难逃死亡的命运——霍金辐射下黑洞逐渐蒸发。

  • 最终宇宙只剩下光子与轻子（注：基于质子衰败的假设），结果及其漫长的时间后达到热寂。

此时，已经是$10^{10000}$年后，而有生命的时段只是其中的一瞬。者太可怕，一个看淡个人生死的人，却在看不淡宇宙的生死。我生起一种深深的恐惧和遗憾，我不希望她就这样死掉，我希望宇宙有来生，重现勃勃生机。可是现在的观测告诉我们，宇宙在加速膨胀，这支持了宇宙将持续膨胀而不是坍缩的理论，因而循环大爆炸指望不上了，宇宙哪来的来生呢。

宇宙的繁衍

正在我恐惧与遗憾之时，这个视频介绍了一种有趣的猜想：

因此，高等科技文明创造小宇宙是宇宙繁衍的途径。而物理定律则是宇宙的基因。在众多宇宙中，物理定律各不同，能孕育智慧生命的宇宙得以繁衍。不能孕育生命的宇宙没有后代。

意义的建立

人类作为一种智慧生命，是这个宇宙繁衍的一个可能。而正在读博的我，是人类的科技发展某个中间阶段的普通工作者。此时此刻，我找到了自己科研的意义，乃至我存在的意义：

人类要想继续发展科技，去创造小宇宙，需要极大的能量，得采集大量的恒星，故而必需成为星际文明。而在星际文明的角逐中，人类这种松散的社会结构、自私内斗的先天习性，会使得人类被淘汰。要克服这个生物进化的历史遗留问题，要么改造人类的基因，去掉自私的成分，延续碳基文明；要么用强人工智能延续人类的文明，组建更团结紧凑的社会结构，走向硅基文明。

而通往强人工智能的慢慢长路上，我们才刚刚起步。仅依靠基于神经网络的视听感官的智能是远远不够的，还需要有符号、推理、逻辑等等这些东西。于是在此起承转合之际，我们这些正在研究这些课题的博士生们啊，真可谓是做着至关重要的工作。我们因参与宇宙繁衍这样的伟大工作而有意义。

想到这里，我突然想回去搞我的科研了。如果有足够长的寿命，搞完人工智能，我还要去再读几个PhD搞基因科学、可控核聚变、量子计算，这都是人类成为靠谱的星际文明前需要完成的。

意义的破除

有朋友问我，这么宏大而远离的此生的目标，真的可以提供意义吗？我再次陷入了沉思，要是你看淡了世间万物的生死，明白了“一切诸行无常，一切法无我，涅槃寂灭” 的道理，又怎会遗憾宇宙的无后呢？既然众生是苦的，那宇宙来生中的众生不也是苦的呢？宇宙连同其中众生一同涅槃不香吗？想到这里，我觉得之前找到的那个存在的意义好荒唐啊，于是我又迷失。

组合规则

  • 始于足下

    九层之台,起于累土。(建字底，廴)；千里之行,始于足下。(走之旁，辶)

    • 以“廴 辶”为部首的字，首码先取z: 如 廷 (tkzr) jrji
    • 以“廴 辶”为部首的字, 依笔顺在其之前其他部首或小字，则末码先取z: 如随 (svez), 挺 (tkfz), 庭 (tkgz)

  • 首末双形

    • 遵循字根笔顺。
      字根按书写顺序排序

    • 取首和末两个形码。
      先取小字或部件，否则取笔画

  • 不拆穿

    • 不要往穿插式构造的方向去拆字，因为结构会被破坏。
      (中，未，末，重，里，……)

  • 不分开

    • 不要往隔开式构造的方向去拆字，因为结构会被破坏。
      (办，半，米，……)

小字

小字衍生规则

  • 小字的判定:

    无法被如下衍生得到的字是元小字

    通过如下方式在小字上加1-3笔衍生的字都是小字

    • 延长: 用所加笔画将小字中的笔画延长

      己->巳, 已

    • 穿插: 存在所加笔画x, 存在小字的笔画y, 使得 去掉端点(x) ∩ 去掉端点(y) ≠ ∅

      中=口+丨，未=一+木，末=一+木, 井=一+廾

    • 隔开: 不是穿插, 且存在小字的笔画y, 使得 所加诸(>=2)笔画的凸包 ∩ 去掉端点(y) ≠ ∅

      办=力+丶+丶, 为=丶+力+丶, 米=丶+丶+木, 来=一+米, 半=丶+丶+一+十……

    • 组合: 以上除外的情况, 且所加笔画

      旦=日+一,良=丶+艮, 只=口+八, 白=丿+日, 失=丿+夫, 大=+乀, 巴=巳+丨

  • 输入小字:

    • 元小字: 首笔画+末笔画

    • 变形/穿插/隔开式小字: 首笔画+末笔画（对应组合规则中的 “不拆穿” “不分开”）

    • 组合式小字:

      • 若组合的方式 与实际笔顺相同, 则根据实际笔顺: 首小字/笔画+末小字/笔画

      • 若组合的方式 与实际笔顺不同, 则: 小字+加的笔画的末笔

        例如: 巴BASL=巳S+丨L

症状

运行 tensorboard --logdir .

报错

tensorboard 8 <module>

main.py 75 run_main

PermissionError:
13
Permission denied

原因

当前用户没有/tmp/.tensorboard-info的读写权限, 故而tensorborad无法写入临时目录

对策

  • 若有root权限

    sudo chmod 777 /tmp/.tensorboard-info
sudo chown root:root /tmp/.tensorboard-info

    然后运行tensorboard --logdir .

  • 若无root权限

    # 申明环境变量
export TMPDIR=/tmp/$USER;
# 创建目录
mkdir -p $TMPDIR;

    然后运行tensorboard --logdir .
